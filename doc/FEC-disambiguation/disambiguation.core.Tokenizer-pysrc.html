   1  ''' 
   2  This module defines a number of classes used to parse and tokenize the data 
   3  within records. Most importantly, the L{Tokenizer} class and its subclasses 
   4  define methods for "normalizing" various record fields such as first name or 
   5  last name. The normalized fields are added to the record as additional dict 
   6  key-values.  
   7   
   8  After normalizing the fields, Tokenizer also "tokenizes" the recods. 
   9  For each field, the value is processed into tokens. Tokenizer defines tokens 
  10  as full "words" separated by whitespace while TokenizerNgram defines it as 
  11  character n-grams. Based on these tokens, for each record, a "feature vector" 
  12  is computed that shows which of the tokens it possesses. These feature vectors 
  13  will later allow us to compute LHS hashes for the records which are used to 
  14  find similar records efficiently. 
  15   
  16  The other class in this module is the L{TokenData} class which is instantiated 
  17  by the Tokenizer class and contains the global data on the tokens discovered 
  18  by Tokenizer such as their frequency, and an index of all tokens.
