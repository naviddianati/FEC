  2  # This program analyzes the full database for a given state. That is, not just the <state>_addresses 
  3  # 
  4  # The <state>_addresses tables are roughly 0.25 the size of their corresponding <state> tables. Why? 
  5  # My explanation is that lots of records that have unique identifier pairs in the <state> table, 
  6  # occur multiple times in the contributor_addresses table, and therefore don't appear in its unique 
  7  # version, namely contributor_addresses_unique. 
  8  ''' 
  9      TODO: 
 10          1. (DONE: <state>_combined) We need a new table that combines <state> and <state>_addresses as follows: 
 11          The table structure is like <state>_addresses, but it contains all the records found in <state>. 
 12          For records that have address fields in <state>_addresses, use those. For others, use null. 
 13   
 14          2. This table will be analyzed in this code. 
 15   
 16          3. Add additional fields to each record: 
 17          Name frequency, Timeline period (if exists), Size of neighborhood in employer graph?, Centrality in Employer graph? Size of neighborhood in occupation graph 
 18   
 19          4. Identify the right set of features of the records from this table to use with Disambiguator. 
 20          Candidates: Name, Address, Employer, Occupation, CMTE_ID (may be confounding) 
 21   
 22          5. Define "possible match". Identifying possible matches will be the first stage of disambiguation. We will refine our knowledge starting from these "possible matches". 
 23          ? Name is similar (specify) 
 24          ? Address is same 
 25          ? Employer is 
 26  ''' 
 27  ''' 
 28  Strategy: 
 29      - Using <state>_addresses, find possible matches (use Disambiguator) 
 30      - Using the results, use AffiliationAnalyzer to derive network of affiliations 
 31      - Query <state>_combined. Contains all records from <state>, and for some, additional data from <state>_addresses 
 32      - Find possible matches using Disambiguator and additional attributes: frequencies, affiliations, etc. 
 33      - A different comparison function is needed. 
 34      -
