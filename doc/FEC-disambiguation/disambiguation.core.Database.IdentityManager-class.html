get_related_identities(self, identity)

source code 

Return a list of all identities related to the given identy. These include any identity some of whose records were compared to the records of the given identity regardless of the (no, maybe, yes) values. The decision of which identities should be considered linked to the given identity must be made by another function using the output of this function.

Parameters:
  • identity - a string. The stage1 identity of a cluster.
Returns:
a dict of other_identity: relation key-value pairs.

get_reachable_identities(self, identity, fcn_linked=None)

source code 

Return all identities that are reachable from identity via a path of edge scores 1 on the graph of related identities. This will be done using the function self.get_related_identities, but not all "related" identities will necessarily make it to the list of linked identities.

Parameters:
  • identity - the identity in question.
  • fcn_linked - a function that decides if two identities that are linked by an edge in self.dict_identity_2_identities are close enough that they should be considered linked. This function will be used to find all identities that are connected to "identity" through a path of such edges. If this function is not provided, a default will be used.
Returns:
a list of all identities that we judge to be linked to the given identity.

generate_dict_identity_adjacency(self, list_identity_pairs, overwrite=False, export_file='', verdict_authority=None)

source code 

Compute self.dict_identity_adjacency from a list containing results of pairwise S1 identity comparisons. This is a stage2 operation.

Parameters:
  • list_identity_pairs - a list where each item is of the form ((identity1,identity2), result). Each item is then the result of a record pair comparison. The tuple (identity1, identity2) is sorted. This is a much simpler operation than the v1 method, since now list_identity_pairs already contains the final comparison results for the S1 identity pairs and no further calculations are required.
  • verdict_authority - an instance of VerdictAuthority that issues a final verdict on an element of list_identity_pairs. This object must be supplied whether this is a bootstrapping or a disambiguation run. The bootstrapping object can be a dummy object with an lambda x:1 as the verdict function.

Compute all the super identities and for each one determine if it has internal middle name conflict. The key is a sorted tuple of all identities reachable from one another. The value is an integer counting the number of middle name conflicts within the set given the edges for which we have comparison results.

get_linked_identities(self, identity)

source code 

Using the table linked_identities, return a list of all other identities linked to identity. In previous versions, we did this just by using self.dict_identities_adjacency but now, that is not enough. We must use the contents of linked_identities which are the result of further processing of the said dict, after resolving middle name conflicts and splitting some of the super identities.

Instance Variable Details [hide private]

dict_identity_2_identities

dict mapping a each identity to others it's related to. Each value is a dict of other_identity: relationship key-value pairs. More convenient to use in place of self.dict_identity_adjacency when we have a single identity and need to find all related identities.

dict_identity_adjacency

dictionary mapping a (sorted) tuple of identity identifiers to a tuple of numerical scores indicating their relationship. The value is a tuple of 3 integers: (inconsistent, mayberelated, related). Each element in this tuple is a weighted count of inter-identity record-pair comparison results. inconsistent counts the number of record pairs between the two identities with a strong inconsistency. mayberelated counts the number of results that aren't conclusive either way but suggest some similarity. related counts the number of results that indicate a clear/strong link.
